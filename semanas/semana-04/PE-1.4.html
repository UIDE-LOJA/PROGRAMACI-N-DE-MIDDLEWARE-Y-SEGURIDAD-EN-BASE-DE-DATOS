Esta actividad práctica te permitirá aplicar los conceptos de seguridad en middleware mediante el análisis de vulnerabilidades y modelado de amenazas. Te enfocarás en identificar vectores de ataque, evaluar riesgos, y proponer contramedidas basadas en los principios CIA.

¿Qué realizarás?

Actividad Instrucciones específicas Entregable
Preparación individual Revisa presentación Semana 4 sobre CIA, modelos de amenaza, STRIDE y criptografía aplicada a APIs. Comprensión de conceptos de seguridad.
Análisis del sistema Selecciona un sistema API REST (puede ser ficticio). Identifica: componentes, flujo de datos, puntos de entrada, activos críticos. Diagrama de arquitectura del sistema.
Modelado de amenazas STRIDE Aplica STRIDE para identificar amenazas: Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege. Tabla con amenazas identificadas.
Evaluación de riesgos Para cada amenaza, evalúa: probabilidad (Alta/Media/Baja), impacto (Alto/Medio/Bajo), nivel de riesgo (Crítico/Alto/Medio/Bajo). Matriz de riesgos completa.
Contramedidas Propón contramedidas para amenazas críticas/altas: autenticación (JWT, OAuth), cifrado (TLS, hash), validación, rate limiting, logging. Tabla de contramedidas con justificación.
Entrega final Compila PDF con diagrama, análisis STRIDE, matriz de riesgos, contramedidas y breve reflexión (máximo 200 palabras) sobre seguridad en APIs. PDF completo con análisis de seguridad.

Ejemplo de análisis STRIDE:

Amenaza STRIDE Descripción Ejemplo en API Afecta CIA
Spoofing Suplantación de identidad Atacante usa token robado Autenticación
Tampering Modificación de datos Man-in-the-middle modifica payload Integridad
Repudiation Negación de acciones Usuario niega haber hecho request No repudio
Info Disclosure Exposición de información API expone datos sensibles sin cifrar Confidencialidad
Denial of Service Denegación de servicio Flood de requests satura servidor Disponibilidad
Elevation of Privilege Escalada de privilegios Usuario normal accede a admin endpoints Autorización

Ejemplo de contramedidas:

  • Spoofing: JWT con firma HMAC-SHA256, OAuth 2.0, MFA
  • Tampering: TLS 1.3, HMAC para integridad de mensajes
  • Repudiation: Logging centralizado, audit trail, timestamps
  • Info Disclosure: Cifrado en tránsito (TLS) y reposo (AES-256)
  • DoS: Rate limiting, throttling, WAF, CDN
  • Elevation: RBAC, validación de permisos, principio de mínimo privilegio
