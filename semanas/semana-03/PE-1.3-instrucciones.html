Esta actividad práctica te permitirá implementar un servicio de eco en Node.js que responda en JSON y Protocol Buffers, aplicando los conceptos de serialización y protocolos de comunicación vistos en la Semana 3. Te enfocarás en crear un servicio funcional, manejar múltiples formatos, y realizar testing de performance.

¿Qué realizarás?

Actividad Instrucciones Entregable
1) Configuración inicial Crea un proyecto Node.js con Express. Instala dependencias necesarias: express, protobufjs (o similar). Proyecto inicializado con package.json.
2) Definición de esquema Define un archivo .proto simple para el mensaje de eco (ej.: EchoRequest con campo "message" y EchoResponse con "message" y "timestamp"). Archivo .proto con esquema básico.
3) Implementación del servicio Crea endpoints POST /echo/json y POST /echo/protobuf que reciban datos y respondan en el formato correspondiente. El servicio debe devolver el mensaje recibido más un timestamp. Código del servicio con ambos endpoints funcionales.
4) Testing básico Prueba ambos endpoints con herramientas como Postman o curl. Verifica que JSON responda correctamente y que Protobuf serialice/deserialice apropiadamente. Capturas de pantalla o logs de pruebas exitosas.
5) Comparación de performance Realiza 10 peticiones a cada endpoint y compara: tamaño de respuesta (bytes) y tiempo de respuesta promedio. Documenta los resultados. Tabla comparativa con métricas de performance.
6) Documentación Incluye README con instrucciones de instalación, ejecución y uso de los endpoints. Explica brevemente las diferencias observadas entre JSON y Protobuf. README.md completo.

Estructura mínima del proyecto:

proyecto-eco/
├── package.json
├── server.js (o index.js)
├── echo.proto
├── README.md
└── tests/ (opcional: capturas o scripts de prueba)

Ejemplo de esquema .proto:

syntax = "proto3";

message EchoRequest {
  string message = 1;
}

message EchoResponse {
  string message = 1;
  int64 timestamp = 2;
}

Criterios de evaluación:

  • Servicio funcional con ambos endpoints (JSON y Protobuf)
  • Serialización/deserialización correcta en ambos formatos
  • Testing documentado con resultados
  • Comparación de performance con métricas concretas
  • Documentación clara y completa
