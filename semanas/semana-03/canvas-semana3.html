Esta semana exploramos protocolos de comunicación modernos y tecnologías de serialización, analizando cuándo usar HTTP/2, WebSockets o gRPC según el contexto. Profundizaremos en servicios de comunicación eficientes y implementaremos soluciones prácticas con JSON y Protocol Buffers.

¿Qué veremos?

Tema Contenido clave Ejemplos
HTTP/2 y Multiplexing Protocolo binario con multiplexing de streams, server push, compresión de headers (HPACK); mejoras de rendimiento sobre HTTP/1.1; cuándo migrar y consideraciones de implementación. APIs REST de alta concurrencia, aplicaciones web modernas, CDNs, servicios de streaming.
WebSockets Comunicación bidireccional full-duplex sobre TCP; conexión persistente para tiempo real; manejo de eventos, heartbeat y reconexión; escalabilidad con load balancers. Chat en tiempo real, notificaciones push, gaming online, colaboración en vivo, trading financiero.
gRPC y Protocol Buffers RPC moderno sobre HTTP/2 con serialización binaria; definición de servicios con .proto; streaming bidireccional; generación automática de clientes; interoperabilidad entre lenguajes. Microservicios internos, APIs de alta performance, comunicación entre servicios, sistemas distribuidos.
Serialización de Datos Comparación JSON vs Protocol Buffers vs MessagePack; eficiencia en tamaño y velocidad; esquemas evolutivos; compatibilidad hacia atrás; casos de uso específicos. APIs públicas (JSON), comunicación interna (Protobuf), IoT (MessagePack), logs estructurados.
Servicios de Eco y Testing Implementación de servicios de prueba que respondan en múltiples formatos; testing automatizado con Postman; scripts de cliente para validación; monitoreo de performance. Servicios de desarrollo, testing de APIs, validación de protocolos, benchmarking de rendimiento.
Selección de Protocolo Criterios de decisión: latencia, throughput, compatibilidad, complejidad; matriz de decisión para HTTP/2 vs WebSockets vs gRPC; consideraciones de infraestructura y equipo. APIs públicas vs internas, aplicaciones tiempo real vs batch, equipos polyglot vs mono-stack.

Objetivos de aprendizaje

Al finalizar la semana podrás:

  • Comparar HTTP/2, WebSockets y gRPC, identificando cuándo usar cada protocolo según requisitos específicos.

  • Implementar servicios que respondan en JSON y Protocol Buffers, manejando serialización eficiente.

  • Crear colecciones de Postman y scripts de cliente para testing automatizado de APIs con diferentes protocolos.
  • Evaluar criterios de selección de protocolos considerando performance, complejidad y casos de uso específicos.
