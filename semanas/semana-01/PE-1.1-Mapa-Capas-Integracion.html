Esta actividad práctica te permitirá aplicar los conceptos fundamentales de sistemas distribuidos mediante el diseño y visualización de arquitecturas multicapa. Te enfocarás en identificar capas lógicas, mapear puntos de integración, y justificar decisiones arquitectónicas usando criterios técnicos.

¿Qué realizarás?

Actividad Instrucciones específicas Entregable
Preparación individual Revisa Compendio Semana 1: conceptos de SD, roles del middleware y estilos arquitectónicos. Comprensión de fundamentos de sistemas distribuidos.
Análisis del caso Analiza el sistema asignado e identifica las capas lógicas (presentación, negocio, datos) y componentes middleware. Identificación clara de capas y componentes del sistema.
Diseño del diagrama Crea un diagrama visual que muestre las capas, componentes y puntos de integración entre ellos. Diagrama arquitectónico completo y legible.
Documentación técnica Completa la matriz de análisis explicando cada capa, sus responsabilidades y puntos de integración. Matriz de análisis con justificaciones técnicas.
Entrega final Presenta diagrama final con matriz completada y reflexión sobre decisiones arquitectónicas tomadas. Diagrama final con documentación técnica completa.

Matriz de análisis arquitectónico a completar

Capa/Componente Responsabilidades Tecnologías/Middleware Puntos de Integración
Capa de Presentación Interfaz de usuario, captura de datos, validación básica Especifica tecnologías frontend utilizadas Describe cómo se conecta con la capa de negocio.
Capa de Negocio Lógica de negocio, procesamiento, reglas APIs, servicios, middleware de aplicación Conexiones con presentación y datos.
Capa de Datos Persistencia, acceso a datos, transacciones SGBD, ORM, middleware de datos Interfaces con capa de negocio.
Middleware de Comunicación Comunicación entre componentes distribuidos HTTP/REST, gRPC, WebSockets, MOM Protocolos y formatos de intercambio.
Middleware de Seguridad Autenticación, autorización, cifrado JWT, OAuth2, TLS, firewalls Puntos de control de acceso.
Servicios Externos Integración con sistemas terceros APIs externas, servicios cloud Interfaces y adaptadores de integración.

Casos de estudio disponibles:

  • Sistema de E-commerce: Plataforma de comercio electrónico con múltiples vendedores
  • Sistema Bancario Digital: Aplicación móvil y web para servicios financieros
  • Plataforma Educativa: LMS con videoconferencias y gestión de contenidos
  • Sistema de Salud: Historia clínica electrónica con telemedicina
  • Red Social Empresarial: Plataforma de colaboración interna
  • Sistema IoT Industrial: Monitoreo y control de dispositivos industriales

Materiales: Compendio Semana 1 - Fundamentos de sistemas distribuidos, roles del middleware y estilos arquitectónicos.

Información importante

  • Modalidad: Práctica de experimentación presencial — 2.25 pts.

  • Duración: 90 minutos.

  • Entrega: Diagrama y matriz completada al final de la clase.

  • Herramientas sugeridas: Papel y lápiz, Lucidchart, Draw.io, o cualquier herramienta de diagramación.
