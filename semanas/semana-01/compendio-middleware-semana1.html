Conceptos de Sistemas Distribuidos

Evolución de los Sistemas Distribuidos

Definición Formal

"Un conjunto de computadores independientes, interconectados a través de una red y con capacidad de colaborar con el fin de realizar una tarea."

Académicamente: "Una red de ordenadores independientes que aparecen como un sistema coherente a sus usuarios"

Ventajas de los Sistemas Distribuidos

Los sistemas distribuidos ofrecen múltiples beneficios que los hacen esenciales en la computación moderna

Navega verticalmente (↓) para explorar cada ventaja

1. Compartición de Recursos

Reflejo de la Arquitectura Organizacional

Los sistemas distribuidos permiten compartir recursos a través de la red, reflejando la arquitectura organizacional moderna donde diferentes departamentos y equipos necesitan acceder a recursos comunes.

Recursos Compartibles
  • Bases de datos centralizadas
  • Servicios de autenticación
  • Sistemas de archivos distribuidos
  • Capacidad de procesamiento
Beneficios Organizacionales
  • Reducción de costos operativos
  • Consistencia de datos
  • Centralización de políticas
  • Mejor colaboración entre equipos

2. Escalabilidad

Superando Limitaciones Monolíticas

La capacidad de incrementar recursos según la demanda es una de las principales ventajas, permitiendo que los sistemas crezcan horizontalmente sin las limitaciones de arquitecturas monolíticas.

Escalabilidad Horizontal
  • Añadir más servidores
  • Distribución de carga
  • Particionamiento de datos
  • Replicación de servicios
Escalabilidad Vertical
  • Aumento de CPU y memoria
  • Mejora de almacenamiento
  • Optimización de red
  • Actualización de hardware

3. Tolerancia a Fallos

Replicación y Disponibilidad

La replicación de recursos asegura la disponibilidad del sistema incluso ante fallos de componentes individuales, proporcionando robustez y confiabilidad.

Estrategias de Replicación
  • Replicación de datos
  • Servicios redundantes
  • Balanceadores de carga
  • Failover automático
Niveles de Disponibilidad
  • 99.9% - 8.76 horas/año downtime
  • 99.99% - 52.56 minutos/año
  • 99.999% - 5.26 minutos/año
  • 99.9999% - 31.5 segundos/año

Roles del Middleware

Definición y Propósito

Middleware (Mw): Capa de software que establece conexión entre el sistema operativo y las aplicaciones, datos y usuarios.

Función Principal

Proporcionar servicios y funciones comunes como Single Sign-On (SSO) o gestión de APIs

Funciones Clave del Middleware

1. Abstracción y Conectividad

El middleware proporciona un conjunto de interfaces de programación de aplicaciones (APIs) "más funcional que el sistema operativo y los servicios de red", permitiendo transparencia de localización, independencia de servicios de red y escalabilidad

2. Integración y Heterogeneidad

El middleware es fundamental para "soportar la comunicación entre procesos a través de plataformas heterogéneas". Las herramientas de integración conectan sistemas mediante transformación, mensajería empresarial y autenticación SSO

3. Desacoplamiento

Opera como "espacio de memoria compartida" permitiendo desacoplamiento temporal y referencial

Tipos de Middleware

El middleware puede tomar diversas formas según las necesidades de comunicación y procesamiento en sistemas distribuidos

TP Monitors

Monitores de Procesamiento de Transacciones

Características Principales

  • Entorno distribuido: Aplicaciones multi-servidor
  • Transacciones ACID: Atomicidad, Consistencia, Aislamiento, Durabilidad
  • Gestión de concurrencia: Control de acceso simultáneo
  • Recuperación ante fallos: Rollback automático
  • Protocolo 2PC: Two-Phase Commit
  • Load balancing: Distribución de carga
  • Monitoring: Supervisión en tiempo real
  • Escalabilidad: Horizontal y vertical

RPC

Llamadas a Procedimientos Remotos

Características Principales

  • Transparencia de ubicación: Llamadas remotas como locales
  • Marshalling automático: Serialización de parámetros
  • Gestión de errores: Timeouts y reconexión
  • Múltiples protocolos: HTTP, gRPC, JSON-RPC
  • Stub generation: Código cliente/servidor automático
  • Load balancing: Distribución de llamadas
  • Service discovery: Localización automática de servicios
  • Monitoring: Métricas y trazabilidad

Navega verticalmente (↓) para explorar la simulación de microservicios

RPC - Simulación de Microservicios E-commerce

MOM

Middleware Orientado a Mensajes

Características Principales

  • Intercambio de datos: Aplicación a aplicación
  • Comunicación asíncrona: Sin bloqueo de procesos
  • Desacoplamiento temporal: Productores y consumidores independientes
  • Persistencia: Mensajes almacenados de forma segura
  • Escalabilidad: Múltiples productores y consumidores
  • Tolerancia a fallos: Garantías de entrega

Tecnologías Populares

RabbitMQ

Ideal para comandos y workflows complejos

Apache Kafka

ORBs

Agentes de Solicitud de Objetos

Características Principales

  • Transparencia de ubicación: Objetos remotos como locales
  • Interoperabilidad: Diferentes lenguajes y plataformas
  • IDL (Interface Definition Language): Definición de interfaces
  • Servicios integrados: Naming, Trading, Security
  • Object Adapter: Gestión del ciclo de vida
  • Interface Repository: Metadatos de interfaces
  • Dynamic Invocation: Llamadas en tiempo de ejecución
  • CORBA/COM/DCOM: Implementaciones estándar

Estilos y Capas Arquitectónicas

Arquitecturas Multicapa

La arquitectura de una aplicación puede referirse a sus capas lógicas (layers), que son "capas de software que nosotros, los desarrolladores/as, creamos". El principio es dividir una aplicación en capas que colaboran mediante interfaces bien definidas

— DWES DOCS, IES Celia Viñas

El diseño más utilizado actualmente es el diseño en tres capas

Ventajas del Diseño Multicapa

  • Separación de responsabilidades
  • Mantenibilidad y escalabilidad
  • Reutilización de componentes
  • Testabilidad independiente

Puntos de Integración entre Capas

Conectividad y Comunicación

Los puntos de integración definen cómo las capas se comunican e intercambian información de manera controlada

Presentación ↔ Negocio

  • APIs REST: HTTP/JSON para operaciones CRUD
  • GraphQL: Consultas flexibles de datos
  • WebSockets: Comunicación en tiempo real
  • RPC/gRPC: Llamadas a procedimientos remotos

Negocio ↔ Datos

  • ORM/ODM: Mapeo objeto-relacional
  • SQL/NoSQL: Consultas directas a BD
  • Stored Procedures: Lógica en base de datos
  • Connection Pools: Gestión de conexiones

Capas vs Niveles

Capa (Layer)

Referencia lógica

Segmentación desde punto de vista lógico (Presentación, Lógica, Datos)

Nivel (Tier)

Referencia física

Distribución física en hardware (ej: 3 capas en 1 máquina = 3 capas, 1 nivel)

Patrón Modelo-Vista-Controlador (MVC)

Caso particular de arquitectura 3 capas con responsabilidades estandarizadas

Modelo

Lógica de negocio, acceso a datos, filtros y restricciones del sistema

Vista

Interfaz de usuario, obtiene datos del Modelo para generar la interfaz

Controlador

Gestiona eventos, accede al Modelo y delega a la Vista

Estilos Arquitectónicos Modernos

Cliente-Servidor (C-S)

Modelo popular en aplicaciones web. Servidor espera peticiones pasivamente, cliente invoca y espera respuestas

Peer-to-Peer (P2P)

Procesos con capacidades idénticas, interacciones directas. Descentralización, distribución, balance de carga

Microservicios

Separación en módulos independientes. API Gateway como punto de entrada, escalabilidad y resiliencia
